@djestick Observer HUD
======================

Кастомный OBS HUD для CS2, подготовленный под бренд `@djestick`. Репозиторий содержит исходники на React/Vite, набор публичных ресурсов и сериализованные конфиги HUD Framework.

Возможности
-----------
- Набор готовых оверлеев с поддержкой радаров, trivia-блоков и превью игроков.
- Гибкая панель управления (`public/panel.json`) и горячие клавиши (`public/keybinds.json`).
- Скрипты сборки и упаковки (`npm run build`, `npm run pack`, `npm run sign`) для быстрой подготовки релиза.
- Совместимость с Bolt Observ + OpenHUD экосистемой.

Быстрый старт
-------------
1. **Установите зависимости**  
   ```powershell
   npm install
   ```
2. **Запустите дев-сервер**  
   ```powershell
   npm run dev
   ```
   По умолчанию Vite поднимает интерфейс на `http://localhost:5173/`. Для работы HUD требуется подключение к GSI серверу.
3. **Соберите прод-бандл**  
   ```powershell
   npm run build
   ```
   Результат попадёт в каталог `dist/`.
4. **Упакуйте релиз в архив**  
   ```powershell
   npm run pack
   ```
   Команда последовательно выполняет build и формирует ZIP в корне проекта.
5. **Подписанный билд (опционально)**  
   ```powershell
   npm run sign
   ```
   Скрипт соберёт проект и создаст архив `signed.zip` с подписью из `sign.js`.

Структура проекта
-----------------
- `src/` — исходники React-компонентов и стили.
- `public/hud.json` — метаданные HUD (`name`, `author`, версия и поддерживаемые возможности).
- `public/panel.json` — описание контролов для панели управления.
- `public/keybinds.json` — список горячих клавиш.
- `public/*.png|webm|ogg` — статические ассеты HUD.
- `build/` — собранная версия, поставляемая вместе с репозиторием (можно пересоздать командой `npm run build`).

Брендинг и настройки
--------------------
- Название HUD: `@djestick Observer HUD` (см. `public/hud.json`, `index.html`).
- Автор: `@djestick`.
- При необходимости обновите миниатюру (`public/thumb.png`) и иконку (`icon.png`).
- Конфиг `public/hud.json` можно расширять полями Bolt/Observer по требованиям турнира.

Рекомендации по работе
----------------------
- Перед коммитами очищайте каталоги с артефактами (`dist/`, `build/`) либо игнорируйте их git-ом.
- Для кастомизации панели используйте `type`, `label` и `action` поля в `public/panel.json`.
- Изменение горячих клавиш выполняется редактированием массива в `public/keybinds.json`.
- Поддерживайте синхронизацию с командой дизайнеров: ассеты можно заменять без пересборки, если файлы совпадают по именам.
