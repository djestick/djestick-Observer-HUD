# @djestick Observer HUD

Кастомный OBS HUD для CS2, подготовленный под бренд `@djestick`. Репозиторий содержит исходники на React/Vite, набор публичных ресурсов и сериализованные конфиги HUD Framework.

## Возможности
- Набор готовых оверлеев с поддержкой радаров, trivia-блоков и превью игроков.
- Гибкая панель управления (`public/panel.json`) и горячие клавиши (`public/keybinds.json`).
- Скрипты сборки и упаковки (`npm run build`, `npm run pack`, `npm run sign`) для быстрой подготовки релизов.
- Совместимость с Bolt Observ и OpenHUD экосистемой.

## Быстрый старт
1. **Установите зависимости**
   ```powershell
   npm install
   ```
2. **Запустите дев-сервер**
   ```powershell
   npm run dev
   ```
   По умолчанию Vite поднимет интерфейс на `http://localhost:3500/`.
3. **Сборка продакшн-бандла**
   ```powershell
   npm run build
   ```
   Результат будет в каталоге `build/`.
4. **Упаковка релиза**
   ```powershell
   npm run pack
   ```
   Команда последовательно сделает build и запишет zip-архив в корень проекта.
5. **Подписанный билд (опционально)**
   ```powershell
   npm run sign
   ```
   Скрипт соберёт проект и создаст архив `signed.zip` с подписью из `sign.js`.

## Структура проекта
- `src/` — исходники React-компонентов и стили.
- `public/hud.json` — метаданные HUD (`name`, `author`, поддерживаемые фичи).
- `public/panel.json` — описание контролов для панели управления.
- `public/keybinds.json` — список горячих клавиш.
- `public/*.png|webm|ogg` — статические ассеты HUD.
- `build/` — собранная версия (можно пересоздать командой `npm run build`).

## Брендинг и кастомизация
- Название HUD: `@djestick Observer HUD` (см. `public/hud.json`, `index.html`).
- Автор: `@djestick`.
- Миниатюра (`public/thumb.png`) и `icon.png` можно заменить на новые изображения.
- Конфиг `public/hud.json` расширяется полями Bolt/Observer под задачи турнира.

## Рекомендации
- Не коммитьте каталоги с артефактами (`build/`, `dist/`) или добавьте их в `.gitignore`.
- Панель управления настраивается через `type`, `label` и `action` в `public/panel.json`.
- Горячие клавиши редактируются в `public/keybinds.json`.
- Для синхронизации с дизайнерами можно заменять ассеты без пересборки, если имя файла не меняется.
